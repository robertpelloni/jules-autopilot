# Handoff Archive — v1.0.32 Go Resilience and Client Hardening

## Summary
This pass ported resilient degraded-mode session handling to Go and hardened the TypeScript client to support both Google and Go property naming conventions.

## Shipped
- Added resilient fallbacks (mock/error sessions) in `backend-go/api/routes.go`.
- Hardened `lib/jules/client.ts` to support dual property mappings.
- Added `X-Jules-Auth-Token` support in Go request-scoped auth.
- Bumped version to `v1.0.32`.

## Why This Matters
This ensures the dashboard remains usable during live API failures when Go is the active runtime and prevents date/status data loss due to property naming mismatches between runtimes.
