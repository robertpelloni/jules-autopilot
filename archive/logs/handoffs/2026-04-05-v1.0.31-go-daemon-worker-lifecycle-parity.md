# Handoff Archive — v1.0.31 Go Daemon/Worker Lifecycle Parity

## Summary
This pass aligned Go daemon/worker lifecycle semantics more closely with the Bun runtime by coordinating boot/start/stop behavior and surfacing worker-running observability.

## Shipped
- Refactored `backend-go/services/queue.go` worker lifecycle to support stop/start instead of a one-way startup path.
- Added `StopWorker()` and `IsWorkerRunning()`.
- Updated `backend-go/main.go` to auto-start daemon + worker only when Keeper is enabled.
- Updated `POST /api/daemon/status` to start/stop both daemon and worker.
- Added worker-running visibility to health and metrics output.
- Bumped version surfaces to `1.0.31`.

## Why This Matters
This closes another subtle but important runtime semantics gap between the Bun and Go runtimes.
