# Handoff Archive — v1.0.19 Go Template CRUD Parity

## Summary
This pass ported template CRUD into the Go backend so the frontend can manage session templates through Go-native `/api/templates` routes.

## Shipped
- Added `GET /api/templates` to the Go API.
- Added `POST /api/templates` to the Go API.
- Added `PUT /api/templates/:id` to the Go API.
- Added `DELETE /api/templates/:id` to the Go API.
- Added Go-side tag-array ↔ string storage normalization for template responses and writes.
- Bumped version surfaces to `1.0.19`.

## Why This Matters
Templates are an actively used frontend surface. Porting them removes another practical Bun-only dependency and extends the Go migration beyond the core autonomy loop into real product workflows.
