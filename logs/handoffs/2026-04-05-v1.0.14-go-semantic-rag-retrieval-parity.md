# Handoff Archive — v1.0.14 Go Semantic RAG Retrieval Parity

## Summary
This pass ported semantic query / retrieval into the Go backend so the Go runtime now owns both indexing and retrieval for the practical memory loop.

## Shipped
- Added `backend-go/services/rag.go`.
- Implemented OpenAI query embedding generation in Go.
- Implemented cosine similarity search across both `CodeChunk` and `MemoryChunk`.
- Added ranked combined retrieval results with origin metadata.
- Added `POST /api/rag/query` to the Go API.
- Extended Go inactivity nudges to inject retrieval-backed `[LOCAL_CONTEXT]` when smart pilot mode is enabled.
- Bumped version surfaces to `1.0.14`.

## Why This Matters
Before this pass, Go could build memory but not fully consume it. After this pass, Go now owns the practical RAG loop instead of depending on the TypeScript daemon for retrieval.
