# Handoff Archive — v1.0.15 Go Lifecycle Event Parity

## Summary
This pass broadened Go-side lifecycle/detail parity for indexing and issue automation so the frontend can understand more Go-originated automation flows directly through typed websocket events.

  - `codebase_index_started`
  - `codebase_index_completed`
  - `issue_check_started`
  - `issue_evaluated`

- Updated the session Keeper feed to render richer metadata for issue/indexing/RAG details.
- Updated the Go queue to emit the new explicit lifecycle events in addition to Keeper logs.
- Bumped version surfaces to `1.0.15`.

## Why This Matters
The Go backend already owned most major autonomy loops. This pass improved how clearly those Go-driven flows are surfaced to the operator UI, which is an important part of practical runtime parity.
