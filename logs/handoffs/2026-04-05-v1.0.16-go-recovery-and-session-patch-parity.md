# Handoff Archive — v1.0.16 Go Recovery and Session Patch Parity

## Summary
This pass added Go-side failed-session recovery/self-healing plus practical session patch/update support.

## Shipped
- Added `UpdateSession` support to the Go Jules client.
- Added `PATCH /api/sessions/:id` to the Go API.
- Added Go recovery handling for `FAILED` sessions in `check_session`.
- Added provider-backed recovery guidance generation with conservative fallback.
- Added recovery messages sent back into failed Jules sessions.
- Added explicit recovery lifecycle events:
  - `session_recovery_started`
  - `session_recovery_completed`
- Bumped version surfaces to `1.0.16`.

## Why This Matters
The Go backend now owns another meaningful autonomy path that used to be more of a Bun advantage: failed-session self-healing with operator-visible lifecycle telemetry.
