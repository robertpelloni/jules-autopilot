# Project Architecture and Patterns: TurntUpToddler (formerly hymnmania)

This project is a highly automated AI pipeline designed to convert public domain MIDI files, MusicXML sheets, and raw physical sheet music into modern, high-fidelity music videos (e.g., Deep House, Synthwave) suitable for platforms like YouTube and TikTok.

## Core Foundational Principles
*   **Continuous Autonomous Execution:** The pipeline is designed to run automatically. In Daemon mode (`--daemon`) or Live Stream mode (`--stream-rtmp`), the system processes new inputs without manual triggers, bypassing errors to keep the pipeline moving ("Don't stop the party").
*   **Frictionless & Robust:** Requires zero manual configuration for basic asset generation but provides fallbacks for external dependencies (FFmpeg errors, OpenAI outages, Replicate rate limits). It caches expensive API calls (like DALL-E image generation using MD5 hashes) and implements idempotency flags (`--skip-render`, `--skip-remake`) so failed operations can be resumed smoothly.
*   **Modular Pipeline:** Decoupled stages allow independent swapping or upgrading:
    1.  **Raw Input & OMR (Optical Music Recognition):** Translates physical sheets or PDFs into MusicXML using the `oemer` library, parsing syllables and timings using `music21`.
    2.  **Audio Synthesis:** Uses `pybind11` to bridge a robust C++ audio engine (`src/engine/HymnPlayer`) around the FluidSynth C API for fast offline rendering of MIDI data.
    3.  **AI Audio Transformation:** Interfaces with Replicate's MusicGen to create stylistic remakes of the raw synthesis. Global tempos are analyzed via `mido` to ensure the AI output strictly adheres to original timing.
    4.  **AI Stem Separation:** Utilizes `demucs` to split the remade track into drums, bass, vocals, and other (melody), enabling granular ducking when mixing TTS vocals.
    5.  **Generative Voice & Lyrics:** OpenAI (GPT-4) crafts lyrics and DALL-E generates album art. ElevenLabs creates realistic human-like singing. Multi-voice harmonization is achieved via high-fidelity pitch shifting using `pyrubberband` and `librosa`, allowing multiple vocal tracks without altering speed.
    6.  **Video Assembly & Streaming:** Uses FFmpeg to construct the final MP4. Subtitles are generated natively from `.mxl` files (bypassing LLM hallucinations) and burned into the video. The system supports dynamic audio-reactive visualizers (e.g., `showwaves`, `avectorscope`) instead of static cover art. It can stream final results to a 24/7 RTMP server.

## Ecosystem Integration
*   **Omni-Workspace Compatibility:** Strictly adheres to global versioning protocols. It relies heavily on local SQLite databases (`src/db.py`) and global configuration files like `VERSION` and `CHANGELOG.md` to ensure continuous tracking.
*   **Submodule Alignment:** The project has undergone an extensive renaming from `hymnmania` to `TUT` (TurntUpToddler) across all configurations, git submodules (`SUBMODULE_INVENTORY.md`), and documentation to maintain structural integrity with its parent repositories.

## Kids Mode Pipeline
*   **Automated Content Generation:** Activated by `kids_mode=True`. The system includes an automated downloader (`ChildrenSongFinder`) that populates empty inputs with curated, child-friendly public domain nursery rhymes (e.g., "Mary Had a Little Lamb", "Itsy Bitsy Spider").
*   **COPPA Compliance & Safety:** When `kids_mode` is enabled, GPT-4 generation prompts are explicitly constrained to produce safe, educational, and positive content. DALL-E prompts generate child-friendly storybook illustrations. YouTube upload metadata strictly enforces `selfDeclaredMadeForKids: True` to conform to child safety policies.

## Key Design Rules & Observations
*   **Parallel Processing Limits:** Uses `concurrent.futures.ThreadPoolExecutor` in `main.py`, specifically capped (`max_workers=min(4, len(midi_files))`) to avoid overwhelming external APIs and hitting HTTP 429 rate limits.
*   **Audio Mixing Precautions:** In `src/utils.py`, when mixing synthesized vocals with the instrumental track, the instrumental is carefully ducked (-3dB) and the vocals boosted (+2dB) to create headroom and prevent digital clipping.
*   **Dependency Management & Testing:** Requires system packages like `fluidsynth` and `libfluidsynth-dev`. Python tests use `pytest` with `pytest-mock` to avoid burning real API credits during CI/CD. C++ engine tests run via `make tests/run_tests`.
*   **Docker Containerization:** Currently utilizes a monolithic multi-stage build, but heavy ML dependencies (`oemer`, `demucs`, OpenCV) create a bloated container. Future architectural optimizations point towards shifting these heavy inferences into separate Alpine or Distroless microservices.
